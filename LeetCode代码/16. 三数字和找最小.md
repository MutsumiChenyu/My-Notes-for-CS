
## Zero-Copy Philosophy

### 双指针法
通过维护两个索引来定义一个逻辑上的子区间，从而避免创建物理上的子区间（切片拷贝）

说人话就是 你要摘出来新的数据结构 就要进行复制这个操作 时间复杂度高
但是双指针只在元数据上面进行移动 来创建一个模拟新数据 只需要操作指针就行课 内存区块里面不会出现你这个新东西

适用范围：不清楚 可能涉及一个list里面的元素问题的时候可以用这个进行操作

#### 标准解法：双指针法

```python
class Solution:
    def threeSumClosest(self, nums: List[int], target: int) -> int:
        # 1. 排序：这是双指针法的前提
        nums.sort()
        n = len(nums)
        
        # 初始化一个“最接近的和”，随便取前三个数即可
        closest_sum = nums[0] + nums[1] + nums[2]
        
        # 2. 固定第一个数 (i)
        for i in range(n - 2):
            # 优化：如果当前数字和上一个一样，跳过以避免重复计算
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            
            # 3. 双指针初始化
            left, right = i + 1, n - 1
            
            while left < right:
                current_sum = nums[i] + nums[left] + nums[right]
                
                # 核心逻辑 A：比较差距，更新结果
                # 如果当前和更接近 target，则更新 closest_sum
                if abs(current_sum - target) < abs(closest_sum - target):
                    closest_sum = current_sum
                
                # 核心逻辑 B：移动指针
                if current_sum < target:
                    # 和太小了，需要大一点 -> 左指针右移
                    left += 1
                elif current_sum > target:
                    # 和太大了，需要小一点 -> 右指针左移
                    right -= 1
                else:
                    # 如果这就等于 target，直接返回（不可能更近了）
                    return current_sum
                    
        return closest_sum
```
